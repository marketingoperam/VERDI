# Обход блокировки my.telegram.org через hosts

Если при создании приложения на https://my.telegram.org/apps появляется **ERROR**, часто помогает прописать IP напрямую в системный файл `hosts`.

## Куда добавлять (Windows)

Файл:
```
C:\Windows\System32\drivers\etc\hosts
```

Открыть **от имени администратора**:
- Блокнот → ПКМ → «Запуск от имени администратора»
- Файл → Открыть → вставить путь выше

Добавить в конец файла строку:

```
149.154.167.220 my.telegram.org
```

Сохранить. Перезапустить браузер (лучше инкогнито).

## После получения api_id и api_hash — УДАЛИТЬ строку

Когда Telegram сменит IP, сайт может перестать открываться. Удалите добавленную строку из `hosts`.

## Важно для вашего номера +1 (США)

Трюк с `149.154.167.220` чаще работает для **российских** номеров (+7).

У вас номер **+13309563469** (США). Telegram сверяет страну номера и IP. Поэтому:

| Ситуация | Что делать |
|----------|------------|
| VPN включён, номер US | Выключить VPN, попробовать снова |
| Сайт не открывается без VPN | Попробовать hosts с `149.154.167.220` |
| ERROR всё равно | Platform: **Desktop**, URL: `https://example.com`, уникальный short name |

На вашей сети DNS сейчас отдаёт для my.telegram.org IP **149.154.167.99** (может отличаться от 220).

## Быстрые скрипты в этой папке

- `add_telegram_hosts.bat` — добавить строку (нужен запуск от администратора)
- `remove_telegram_hosts.bat` — убрать строку (тоже от администратора)

## Когда получите ключи — пришлите

```
api_id: ...
api_hash: ...
```

Номер +13309563469 уже сохранён в `.env`.
